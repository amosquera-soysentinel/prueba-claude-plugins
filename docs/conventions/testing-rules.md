# testing-rules.md

## Datos de prueba

- Los datos de pruebas unitarias deben ser coherentes con la información requerida por el dominio.
- No usar datos basura o sin sentido si afectan la comprensión.
- Los valores de ejemplo deben ser legibles y realistas para el caso de uso.

## Reglas operativas para IA

- No generar mocks con textos aleatorios sin contexto.
- No usar valores absurdos si hay una alternativa entendible.
- Los ejemplos deben parecer datos reales del sistema.
